# 新需求落地计划

## Summary

本轮按“先把核心对战助手体验补强，再做轻量首页氛围”处理。优先级为：图鉴与队伍编辑可用性、严格 learnset 数据、伦琴猫顶部伙伴层、知识 tips 轮播。当前不新增第 6 个底部 Tab，仍保持 5 Tab 结构。

## Progress 2026-05-02

- [x] Learnset 第一阶段：181 只已接入基础形态 Pokémon，528 个招式，11323 条 Pokémon-招式关系。
- [x] 图鉴 Pokémon 列表按全国图鉴编号排序；详情头像可点开大图。
- [x] 图鉴详情页当前规则可学招式默认折叠，并支持按属性 / 性质 / 威力排序。
- [x] 队伍小卡片展示当前携带物图片。
- [x] 队伍编辑页招式可搜索，招式行展示属性、性质、威力 / 命中 / PP、简短说明。
- [x] 队伍编辑页携带物可搜索，展示本地道具图片和效果摘要。
- [ ] 伦琴猫顶部伙伴层。
- [ ] 知识问答或 tips 轮播。

## Key Changes

- 图鉴 Pokémon 体验
  - Pokémon 列表显式按全国图鉴编号升序排列；同一 Pokémon 下基础形态在前，Mega / 其他形态跟随基础形态。
  - Pokémon 详情页头像可点击，打开大图预览层，展示当前形态大图，支持关闭返回详情。
  - “当前规则可学会招式”去掉“示例待补齐”口径，改为严格来源数据；默认折叠为下拉/展开列表。
  - 招式列表提供排序控件：按属性、性质、威力排序；性质指 `Physical / Special / Status`。
  - 招式行展示属性 badge、性质、威力、命中、PP、目标范围和短效果说明。

- Learnset 数据
  - 主数据源使用 PokéBase Champions，抽取当前 Reg M-A Pokémon 页面中的可学招式与招式字段。
  - 保留 `manual-review`/sourceRefs 口径，不把 PokéBase 标成官方源；新增审计确保图鉴详情不再展示空的 seed 示例列表。
  - PokeAPI 只用于字段补充或交叉校验，不作为 Champions learnset 主判断源。

- 队伍页与编辑页
  - 队伍小卡片显示当前携带物图片：紧凑卡以小图标叠在 Pokémon 头像附近，展开卡显示图标 + 道具名。
  - 队伍编辑页将招式原生 select 改成可搜索选择器，每个招式槽独立选择。
  - 招式选择器展示属性 badge、性质、威力/命中/PP、简短说明，并只列当前 Pokémon 当前规则可学招式。
  - 携带物选择器改为可搜索，展示道具图片、中文/英文名、效果摘要；仍按当前 Pokémon 过滤 Mega Stone 可用性。

- 伦琴猫与主页轻量内容
  - 不新增首页 Tab；在现有顶部区域做“伦琴猫伙伴层”，随底部 Tab 切换不同姿势。
  - 使用 AI 生成多张透明背景伦琴猫姿势图，放入项目静态资源；姿势包括搭框、探头、站立/趴伏等，按 Tab 映射。
  - 组队页加入轻量“你知道吗”文案轮播，不做答题计分小游戏；内容先来自本地静态 tips，后续可扩展为问答题库。

## Test Plan

- Vitest
  - 图鉴 Pokémon 列表排序：编号升序，Mega 跟随基础形态。
  - 点击详情头像打开/关闭大图预览。
  - 招式排序按属性、性质、威力结果稳定。
  - 队伍小卡显示当前携带物图片。
  - 队伍编辑招式/道具搜索能按中文名、英文名命中，并保存选择。
  - Learnset audit：当前规则内 Pokémon 详情使用 PokéBase 抽取结果，不再依赖旧 seed 示例提示。

- Playwright / visual
  - 更新图鉴详情、队伍紧凑卡、队伍编辑页、组队页顶部伦琴猫与 tips 轮播截图。
  - 使用本机 Chrome channel 跑 `npm run test:pwa` / `npm run test:visual`。

## Assumptions

- PokéBase Champions 可作为本轮主数据源，但仍标为社区/复核级来源。
- “性质”按招式分类 `Physical / Special / Status` 实现。
- “主页小游戏”本轮收敛为“你知道吗”文案轮播，不做交互答题。
- 伦琴猫多姿势接受 AI 生成图片；图片进入项目静态资源，不热链。
