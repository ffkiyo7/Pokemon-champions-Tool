# Pokémon Champions 对战助手 PRD

## 1. 文档信息

- 产品名称：Pokémon Champions 对战助手
- 文档类型：产品需求文档（PRD）
- 目标版本：v1.0 MVP
- 首发平台：移动端 Web / PWA
- 主要语言：中文优先，英文名保留
- 目标规则集：Pokémon Champions Regulation Set M-A / Reg M-A
- 官方规则主源：[Regulation Set M-A 官方公告](https://news.pokemon-home.com/en/page/751.html)

## 2. 产品背景

Pokémon Champions 的规则、可用 Pokémon、Mega 形态、道具与对战机制会随 regulation 更新。对个人玩家来说，组队、查资料、算伤害、调速度线往往需要在多个工具和资料源之间切换，移动端体验也容易碎片化。

本产品希望做一个以“当前 Pokémon Champions regulation”为中心的移动端对战助手。它不是全国图鉴，也不是 Pokémon Showdown 的替代品，而是帮助玩家围绕当前规则快速完成队伍编辑、伤害计算、速度线判断、基础配队检查和合法内容查询。

v1 优先服务个人使用场景，不依赖登录，不做社区分享，不做完整自动组队。核心目标是：数据边界清楚、计算结果可信、移动端使用顺手。

## 3. 产品定位与目标

### 3.1 产品定位

一个面向个人玩家的 Pokémon Champions 移动端对战助手，所有内容默认围绕当前 regulation 展示和计算。

### 3.2 v1 产品目标

- 帮助用户在移动端保存和调整多支队伍。
- 帮助用户基于当前队伍或手动配置完成伤害计算。
- 帮助用户通过速度线可视化判断“快过谁 / 慢于谁”。
- 帮助用户浏览当前规则合法的 Pokémon、招式、道具、特性。
- 帮助用户获得基础配队分析提示，而不是直接替用户生成完整队伍。
- 帮助用户确认当前数据版本与规则来源，降低错误数据带来的决策风险。

### 3.3 非目标

- 不做全国图鉴。
- 不做 Pokémon Showdown 替代品。
- 不做 AI 聊天助手。
- 不做自动生成完整队伍。
- 不做排位战绩记录。
- 不做云同步和账号系统。
- 不做队伍分享广场。
- 不做微信小程序。
- 不做原生 Swift App。

## 4. 目标用户与核心场景

### 4.1 目标用户

- 熟悉宝可梦对战基础、想围绕当前 Champions 规则组队的个人玩家。
- 需要在手机上快速查当前规则、调队伍、算伤害、看速度线的玩家。
- 不希望维护复杂账号系统，只想本地保存个人队伍和常用 benchmark 的玩家。

### 4.2 核心场景

- 新规则更新后，用户查看当前规则、数据版本和可用内容范围。
- 用户创建队伍，按 regulation 限制选择 Pokémon、形态、招式、道具、特性。
- 用户从队伍中选择攻防双方，快速得到伤害范围与击杀概率。
- 用户调整速度投入、性格、Mega、道具、顺风或能力等级，观察速度线位置变化。
- 用户浏览图鉴，从 Pokémon 页面进入伤害计算、速度线或加入队伍。
- 用户刷新数据、导入导出本地队伍，控制个人数据。

## 5. 当前规则集说明

### 5.1 已知官方规则元信息

基于 Pokémon HOME 官方 Regulation Set M-A 公告，PRD 中可确认以下规则元信息：

- 规则名称：Regulation Set M-A
- 规则周期：2026-04-08 02:00 UTC 至 2026-06-17 01:59 UTC
- Eligible Pokémon：以官方公告中 Eligible Pokémon 页面展示为准
- Mega 机制：部分 Pokémon 持有 Mega Stone 时可 Mega Evolution
- Mega 次数限制：每场对战只能使用一次 Mega Evolution
- 道具限制：不允许重复携带相同道具
- 计时：
  - Total Time：20 分钟
  - Player Time：7 分钟
  - Turn Time：45 秒
  - Preview Time：90 秒

### 5.2 不在 PRD 中写死的数据

以下数据不得在 PRD 中凭印象补全，也不在文档中硬编码为最终事实：

- 当前合法 Pokémon 完整列表与数量
- 当前合法招式完整列表
- 当前合法道具完整列表
- 当前可学习招式关系
- Pokémon Champions 特有 Stat Points / IV / EV / Nature 机制细节
- @smogon/calc 或其他计算库对 Pokémon Champions 的支持程度
- 社区使用率和热门 benchmark 的完整排序

这些数据应进入数据维护流程，由权威来源或稳定社区来源确认后版本化管理。

### 6.4 机制未确认时的产品策略

v1 必须避免为了跑通流程而输出伪准确结果。凡是依赖 Pokémon Champions 机制但尚未确认的数据或公式，都按以下策略处理：

- 未确认机制不输出正式计算结论，只展示“机制待确认，无法计算”或“示例数据”。
- 可展示 UI 壳、交互路径和样例值，但必须明确标注“示例数据 / 非真实计算”。
- 伤害计算缺少关键机制时，不展示一确、二确、乱数等最终判断。
- 速度线 v1 可先支持基础速度值和已确认的速度修正；复杂状态、特殊机制和 Stat Points 细节作为待确认项。
- 图鉴和队伍合法性校验只对已确认数据给出强结论；未确认字段使用“需要复核”状态。
- 所有示例值不得进入导出数据，也不得作为正式数据版本发布。

## 7. 平台策略

### 7.1 v1 选择移动端 Web / PWA 的原因

- 验证速度快：适合先验证队伍编辑、伤害计算、速度线和图鉴的产品闭环。
- 分发门槛低：用户可直接通过链接访问，也可添加到主屏幕。
- 迭代成本低：规则和数据变化频繁，Web / PWA 更适合快速修正。
- 本地优先：PWA 可支持缓存与离线访问，满足个人工具的基础需求。
- 未来可迁移：如果 v1 验证成功，可用 Capacitor 包装为 iOS App。

### 7.2 v1 不做微信小程序的原因

- 小程序生态对游戏 IP、图片资源、数据更新、分享路径和审核存在额外不确定性。
- 本产品首要目标是验证对战工具体验，而不是获取微信生态流量。
- 小程序会增加适配和发布成本，稀释 v1 对核心功能的投入。

### 7.3 v1 不做原生 Swift App 的原因

- 原生 App 首发成本高，发布审核和热更新限制不利于快速修正规则数据。
- v1 还需要验证核心交互是否成立，不适合过早承担原生开发成本。
- PWA 若验证成功，再用 Capacitor 或原生方案承接高频用户更稳妥。

### 7.4 后续过渡到 iOS App

- v1 保持移动端页面结构、手势、离线缓存和本地数据模型稳定。
- 当留存和高频使用得到验证后，用 Capacitor 包装，补充 iOS 图标、启动页、离线策略和系统级存储能力。
- iOS App 阶段再评估通知、iCloud / 云同步、分享扩展等能力。

## 8. 信息架构

移动端底部 Tab 建议为：

1. 组队
2. 计算
3. 速度线
4. 图鉴
5. 设置

首页 / 当前规则页不作为底部 Tab。应用启动默认进入“组队”Tab，并在组队页顶部固定展示当前规则卡片；点击规则卡片进入“当前规则详情页”。规则更新、数据过期或首次无缓存时，可以弹出或跳转到当前规则详情页，但底部导航仍保持 5 个核心工具 Tab。

## 9. 页面需求

### 9.1 首页 / 当前规则页

页面目标：

- 让用户确认当前正在使用的 regulation 与数据版本。
- 作为组队页顶部规则卡片和规则详情页，帮助用户理解当前数据范围与规则状态。

核心组件：

- 当前规则卡片：规则名称、有效期、对战类型、是否允许 Mega、道具重复限制、数据版本。
- 数据状态卡片：数据更新时间、来源摘要、是否可刷新。
- 规则详情入口：官方来源、数据版本、已缓存状态、刷新入口。
- 风险提示：当数据版本过旧或规则源未确认时展示。

用户操作流程：

- 打开应用。
- 默认进入组队 Tab，并在页面顶部看到当前规则卡片。
- 点击规则卡片进入当前规则详情页。
- 如提示数据过期，点击刷新数据。

关键状态：

- 数据已同步
- 数据可更新
- 数据过期
- 离线缓存可用
- 首次使用无缓存

空状态：

- 首次打开且无规则缓存时，提示需要联网获取当前规则数据。

错误状态：

- 获取规则失败：展示当前可用缓存版本和重试入口。
- 官方源不可用：提示稍后重试，不使用未确认数据覆盖当前版本。

v1 必须做：

- 展示当前规则名、有效期、Mega 允许情况、数据版本。
- 在组队页顶部固定展示当前规则卡片。
- 提供当前规则详情页，用于查看来源、缓存和刷新状态。
- 展示数据更新时间和刷新入口。

v2 可延后：

- 多规则集切换。
- 规则更新历史。
- 更细的官方公告变更 diff。

### 9.2 组队页

页面目标：

- 支持用户创建、编辑、保存多支当前规则下的队伍。

核心组件：

- 队伍列表
- 队伍详情
- 6 个队伍成员槽位
- Pokémon 选择器
- 形态 / Mega 形态选择器
- 特性、道具、招式、性格、等级、Stat Points 输入区
- 备注区
- 队伍基础分析入口
- 导入 / 导出入口

用户操作流程：

- 用户点击新建队伍。
- 输入队伍名称。
- 添加第 1 只 Pokémon。
- 选择形态、特性、道具、招式、性格、等级和 Stat Points。
- 重复添加至最多 6 只。
- 保存队伍。
- 从队伍成员进入伤害计算或速度线。

关键状态：

- 无队伍
- 队伍编辑中
- 队伍已保存
- 队伍存在非法配置
- 队伍存在缺失字段
- 当前数据版本下配置需要复核

空状态：

- 无队伍时展示“新建第一支队伍”。
- 队伍槽位为空时展示“添加 Pokémon”。

错误状态：

- 当前 Pokémon 不在当前 regulation 合法范围内。
- 招式、道具、特性与当前 Pokémon 或形态不匹配。
- 队伍超过 6 只。
- 道具重复，且当前规则不允许重复道具。
- Stat Points 字段因机制未确认而无法校验时，展示“待机制确认”的弱提示。

v1 必须做：

- 本地创建、编辑、保存多支队伍。
- 每支队伍最多 6 只。
- 所有选择默认按当前 regulation 限制。
- 支持队伍成员备注。
- 支持导出和导入用户队伍数据。

v1.1 / 待调研：

- Showdown paste 导入 / 导出。
- Champions 专属 paste 格式。

v2 可延后：

- 云同步。
- 队伍分享。
- 热门模板导入。

### 9.3 伤害计算页

页面目标：

- 让用户从队伍或手动配置中选择攻防双方，获得清晰、可追溯的伤害计算结果。

核心组件：

- 攻击方配置区
- 防守方配置区
- 招式选择区
- 战斗条件区
- 结果卡片
- 计算说明
- 数据版本提示

用户操作流程：

- 用户从队伍中选择攻击方。
- 用户从队伍中选择防守方，或手动选择防守对象。
- 选择招式。
- 调整战斗条件，例如单打 / 双打、分散伤害、天气、场地、能力等级变化、Mega 状态。
- 查看伤害范围、百分比范围、击杀概率、一确 / 二确 / 乱数说明。
- 根据结果返回队伍页调整配置。

关键状态：

- 攻防双方完整
- 攻击方缺招式
- 防守方缺关键参数
- 战斗条件已修改
- 计算结果来自当前数据版本

空状态：

- 未选择攻击方或防守方时，提示从队伍选择或手动配置。

错误状态：

- 缺少计算必要字段。
- 当前招式与 Pokémon 不匹配。
- 当前规则下该道具或形态不合法。
- 计算机制尚未确认，结果不可输出。
- 数据版本与队伍创建版本不一致，需要复核。

v1 必须做：

- 支持队伍内攻防双方选择。
- 支持手动选择攻击方、防守方、招式、形态、特性、道具、性格、等级、Stat Points。
- 支持单打 / 双打、分散伤害、天气、场地、能力等级变化、Mega 状态。
- 输出伤害范围、百分比范围、击杀概率、一确 / 二确 / 乱数、关键说明。

v2 可延后：

- 多目标批量计算。
- matchup 分析。
- 复杂 AI 解释。

### 9.4 速度线页

页面目标：

- 让用户在调整某只 Pokémon 的速度相关配置时，直观看到它相对于当前规则环境中关键 benchmark 的位置变化。
- 该页面只回答“我现在快过谁 / 慢于谁”，不表达强弱排名。

核心组件：

- Pokémon 选择器
- 速度参数控制区
- 横向速度轴
- 当前 Pokémon 标记
- benchmark 标记
- benchmark 筛选器
- benchmark 详情面板
- 收藏入口
- 数据版本提示

速度参数：

- Pokémon
- 形态 / Mega 形态
- 性格
- Stat Points / 速度投入
- 等级，默认 50
- 道具 / 状态
- 顺风
- 能力等级变化
- 其他 Pokémon Champions 特有速度机制，列为待确认项

坐标定义：

- X 轴为最终速度实数值 / Lv.50 实战速度值。
- Y 轴仅用于视觉分层，例如 benchmark 分类、队伍成员、热门速度档。
- 不做二维强弱地图。
- 当前 Pokémon 的头像或标记根据最终速度值实时移动。
- benchmark 被超过时，高亮或变色。

benchmark 类型：

- 系统预设：当前规则下典型速度线。
- 用户收藏：用户自己关心的对速对象。
- 当前队伍：用户队伍内成员互相对比。

benchmark 筛选：

- 当前队伍成员
- 收藏 benchmark
- 当前环境常见 benchmark
- 全部当前合法 Pokémon 的基础速度线

默认展示策略：

- 默认展示“当前队伍成员 + 收藏 benchmark + 少量系统预设”。
- 不默认展开全部合法 Pokémon，避免移动端信息过载。
- 当用户切换到“全部当前合法 Pokémon 基础速度线”时，需要提示该视图信息密度较高。

benchmark 详情：

- 名称
- Pokémon
- 形态
- 性格
- 速度投入
- 道具 / 状态
- 是否 Mega
- 最终速度值
- 标签，例如“满速”“常见控速”“顺风后”“围巾 / 等效加速”
- 来源
- 备注

用户操作流程：

- 用户选择一只 Pokémon。
- 调整性格、Stat Points、形态 / Mega、道具、顺风、能力等级。
- 当前 Pokémon 标记在速度轴上实时移动。
- 用户观察已超过和未超过的 benchmark。
- 用户点击某个 benchmark 查看配置和说明。
- 用户收藏 benchmark。
- 用户返回队伍页调整队伍配置。

关键状态：

- 已选择 Pokémon
- 参数变化后实时更新
- benchmark 被超过
- benchmark 未被超过
- benchmark 已收藏
- 当前数据源未确认

空状态：

- 未选择 Pokémon 时，展示选择入口。
- 无收藏 benchmark 时，提示可从系统 benchmark 或详情页收藏。
- 队伍为空时，当前队伍成员筛选展示空状态。

错误状态：

- 速度计算必要机制未确认。
- 当前 Pokémon 或形态不在当前 regulation 合法范围内。
- benchmark 来源缺失或数据版本不匹配。
- 当前筛选条件下 benchmark 过多，提示缩小筛选范围。

v1 必须做：

- 实时展示当前 Pokémon 最终速度位置。
- 展示系统预设、用户收藏、当前队伍三类 benchmark。
- 支持点击 benchmark 查看详情。
- 支持收藏 benchmark。
- 控制默认信息密度。

v2 可延后：

- 社区 benchmark 库。
- 使用率驱动的 benchmark 排序。
- 多 Pokémon 同屏对比。
- 自定义速度区间注释。

### 9.5 基础配队分析

页面目标：

- 给用户提供可解释的规则型队伍检查，帮助发现明显短板。
- v1 不直接强行替换 Pokémon，不做 AI 聊天式建议。

核心组件：

- 队伍概览
- 属性弱点分布
- 抗性 / 免疫分布
- 物攻 / 特攻倾向
- 速度覆盖分析
- 功能位检查
- 重复定位提示
- 问题式建议文案

分析项：

- 属性弱点分布
- 抗性 / 免疫分布
- 物攻 / 特攻倾向

- 队伍不足 2 只时，仅给出轻量提示。
- 队伍信息完整时，展示完整分析。
- 队伍存在非法配置时，先提示修正配置。

v1 必须做：

- 输出可解释文案。
- 不生成完整替换队伍。

## 10. 用户流程

### 10.1 新建一支队伍

1. 用户进入组队 Tab。
2. 点击“新建队伍”。
3. 输入队伍名称。
4. 点击空槽位添加 Pokémon。
5. 从当前 regulation 合法 Pokémon 中选择目标。
6. 选择形态 / Mega 形态、特性、道具、招式、性格、等级和 Stat Points。
7. 添加备注。
8. 重复添加至最多 6 只。
9. 保存队伍。
10. 系统提示保存成功，并展示基础配队分析入口。

### 10.2 从队伍进入伤害计算

1. 用户进入某支队伍详情。
2. 选择一名队伍成员。
3. 点击“进入伤害计算”。
4. 系统将该成员带入攻击方或防守方。
5. 用户选择另一方对象。
6. 用户选择招式和战斗条件。
7. 系统输出伤害范围、百分比范围、击杀概率、一确 / 二确 / 乱数说明。
8. 用户根据结果返回队伍调整配置。

### 10.3 调整速度投入并查看速度线变化

1. 用户进入速度线 Tab。
2. 选择一只 Pokémon，或从队伍成员进入速度线。
3. 用户调整性格、Stat Points、形态 / Mega、道具、顺风、能力等级。
4. 当前 Pokémon 标记在速度轴上实时移动。
5. 系统高亮已被超过的 benchmark。
6. 用户点击 benchmark 查看配置、速度值、来源和说明。
7. 用户决定是否收藏该 benchmark。

### 10.4 从图鉴把 Pokémon 加入队伍

1. 用户进入图鉴 Tab。
2. 搜索或筛选 Pokémon。
3. 进入 Pokémon 详情页。
4. 查看属性、种族值、可用特性、可用招式、Mega 信息。
5. 点击“加入队伍”。
6. 选择目标队伍和空槽位。
7. 系统将 Pokémon 加入队伍，并提示补全招式、道具、性格和 Stat Points。

### 10.5 收藏一个速度 benchmark

1. 用户进入速度线 Tab。
2. 切换至系统预设或当前环境常见 benchmark。
3. 点击某个 benchmark。
4. 查看 benchmark 详情。
5. 点击收藏。
6. benchmark 进入用户收藏列表。
7. 后续默认速度线视图展示该收藏 benchmark。

### 10.6 刷新当前 regulation 数据

1. 用户进入首页或设置页。
2. 查看当前规则集版本和数据更新时间。
3. 点击“刷新数据”。
4. 系统请求官方主源和配置的数据源。
5. 刷新成功后更新数据版本。
6. 系统检查本地队伍是否存在因规则更新导致的非法配置。
7. 如存在问题，在队伍列表和队伍详情中提示用户复核。

- Pokémon Champions 官方页面 / 官方规则公告：作为当前 regulation、合法性、规则机制的最高优先级来源。

- 图片资源：必须确认授权或替换为可授权资源，不默认可商用或可公开复制。

## 13. 非功能需求

- 移动端优先：所有核心流程必须适配单手操作和窄屏阅读。
- 离线可用：至少已缓存规则数据、图鉴数据、队伍数据、收藏 benchmark 可离线查看。
- 数据更新可控：自动或手动刷新都不能静默覆盖有风险的数据版本。
- 首屏快：首次进入应优先展示当前规则状态和核心入口。
- 计算准确：伤害计算和速度计算必须基于明确的数据版本与机制假设。
- 结论可追溯：计算结果、分析提示、benchmark 都需要能追溯到数据版本。
- 用户数据本地优先：v1 不依赖登录，不因无账号影响核心体验。
- 错误可恢复：数据刷新失败时保留本地可用版本。
- 信息密度可控：速度线和图鉴筛选默认避免一次性展示过多信息。

### 13.1 v1 验收标准

- 首屏主要内容在正常网络下 2 秒内可见；离线时 2 秒内展示最近一次缓存状态。
- 离线时可查看最近一次缓存的队伍、图鉴基础数据和收藏 benchmark。
- 每个伤害计算结果、速度 benchmark、合法性判断都必须展示或可追溯到数据版本。
- 速度线默认展示 benchmark 不超过 12 个；进入“全部基础速度线”前需要用户主动切换筛选。
- 速度线图表文字不得与当前 Pokémon 标记、benchmark 点、坐标轴边缘发生明显重叠。
- 图鉴搜索无结果时必须提供清除筛选或返回全部列表入口。
- 数据刷新失败时保留当前可用缓存，不清空队伍和收藏数据。
- 队伍页不展示 6 个大型空槽；已填成员后只展示一个紧凑“添加成员”入口。
- 机制待确认时必须展示待确认状态，不输出正式计算结论。

## 14. 风险与待确认项

- Pokémon Champions 当前数据生态仍在快速变化。
- 官方规则与 Showdown / 社区数据可能不同步。
- @smogon/calc 对 Champions 支持程度需要验证。
- Stat Points / IV / EV / Nature 等机制需要确认。
- Mega、道具、招式合法性需要版本维护。
- 中文名、图片、简介文本存在授权问题。
- 速度 benchmark 的选取标准容易主观化。
- 移动端速度线信息密度过高，可能影响可读性。
- Showdown paste 或 Champions paste 格式可能尚未稳定。
- 社区使用率数据可能缺少稳定公开来源。
- 手动维护数据存在过期风险，需要版本提示和复核流程。

- 支持规则数据刷新失败时继续使用本地缓存。

### 18.2 队伍构建

- 支持创建、编辑、删除、保存多支队伍。
- 支持最多 6 名队伍成员。
- 支持 Pokémon、形态、特性、道具、招式、性格、等级、Stat Points、备注字段。
- 支持基于当前规则数据进行合法性提示。
- 支持本地队伍导入 / 导出。

### 18.3 图鉴

- 支持 Pokémon、招式、道具、特性四类内容浏览。
- 支持搜索和基础筛选。
- 支持详情页展示当前规则合法信息和数据版本。
- 支持从 Pokémon 详情加入队伍、进入速度线、进入伤害计算。

### 18.4 伤害计算

- 支持从队伍带入攻击方和防守方。
- 支持手动配置攻防双方和招式。
- 支持关键战斗条件输入。
- 输出伤害范围、百分比范围、击杀概率、一确 / 二确 / 乱数和关键说明。
- 对未确认机制展示“待确认”状态，不输出伪准确结果。

### 18.5 速度线

- 支持选择 Pokémon 并调整速度相关参数。
- 支持实时计算最终速度并映射到横向速度轴。
- 支持系统预设、用户收藏、当前队伍三类 benchmark。
- 支持 benchmark 详情和收藏。
- 默认限制信息密度，避免全部速度线同时铺开。

### 18.6 基础配队分析

- 支持属性弱点、抗性 / 免疫、攻防倾向、速度覆盖、功能位和重复定位检查。
- 输出可解释的问题提示。
- 不自动替换 Pokémon，不生成完整队伍。

### 18.7 设置与本地数据

- 支持查看规则集版本和数据更新时间。
- 支持手动刷新数据。
- 支持清除本地缓存。
- 支持导入 / 导出用户队伍数据。
- 支持中文优先、英文名保留。

### 18.8 数据契约与计算依赖

- 明确每个数据对象的来源、更新时间、数据版本和验证状态。
- 合法性判断必须区分“合法”“非法”“需要复核”“机制待确认”。
- 伤害计算依赖项包括基础数值、招式参数、属性相性、特性影响、道具影响、天气、场地、能力等级变化、Mega 状态和 Champions 特有机制。
- 速度计算依赖项包括基础速度、等级、性格、Stat Points、形态 / Mega、道具、顺风、能力等级变化和其他速度修正机制。
- 在 @smogon/calc 或其他计算依赖未确认前，不把计算结果标记为正式结论。
- 数据导入 / 导出格式需要包含 ruleSetId、dataVersionId 和成员配置版本，避免未来规则更新后无法复核。

## 19. 前端原型交付范围

前端原型用于验证移动端信息架构、页面层级和核心交互，不要求实现真实计算和真实数据同步。

必须交付的屏幕：

- 组队页：当前规则卡片、队伍列表、队伍成员卡、紧凑添加成员入口、基础分析提示。
- 伤害计算页：突出显示伤害范围、一击击倒率、二击击倒率、结论、攻防双方和战斗条件。
- 速度线页：具有坐标层次的速度轴、当前 Pokémon 标记、benchmark 点、benchmark 列表和详情入口。
- 图鉴页：Pokémon / 招式 / 道具 / 特性四类入口，至少展示 Pokémon 详情页。
- 设置页：数据版本、手动刷新、本地缓存、导入导出、语言偏好。
- 当前规则详情页：Regulation Set M-A 元信息、官方来源状态、缓存状态和刷新入口。

每屏需要覆盖的状态：

- 默认态
- 空态
- 错误态或待确认态
- 离线缓存态
- 数据版本可追溯态

假数据规则：

- 原型可使用少量真实 Pokémon 名称和正确图像作为占位，但不得暗示完整合法列表。
- 所有伤害数值、速度值、benchmark 和合法性结论必须标注为示例或可追溯到模拟数据版本。
- Pokémon 图像必须与展示对象匹配；若无法匹配，使用中性剪影或属性图标，不使用错误角色图。

核心演示重点：

- 速度线是原型重点，必须展示“当前速度如何相对 benchmark 移动”的视觉反馈。
- 伤害计算结果卡是计算页重点，视觉权重高于攻防双方和条件控件。
- 队伍页必须避免大型空槽堆叠，已填成员后只保留一个紧凑添加入口。
